# ThriftVerse 👗♻️

**ThriftVerse** is a digital platform for buying, selling, swapping, and discovering secondhand fashion in a more interactive and trusted way. Unlike traditional e-commerce platforms, ThriftVerse combines e-commerce, social interaction, creative fashion content, and sustainable shopping behavior — all in one place.

Built to address the growing fast fashion crisis, ThriftVerse empowers Gen Z and young adults to thrift smarter, connect with sellers directly, and participate in a more sustainable fashion ecosystem.

## 🔥 Key Features

- 🛍️ **Buy & Sell Thrift Items** — Browse and purchase curated secondhand fashion across categories: Men, Women, Kids, Accessories, Shoes.

- 🔁 **Barter System** — Users can offer their own listed products as a trade, with optional partial cash payment if item values differ.

- 🎬 **ThriftVid (Short-Form Videos)** — Sellers can showcase thrift outfits, styling ideas, and product previews in a TikTok-style short-form video feed with comments.

- 💬 **In-App Chat** — Buyers and sellers can chat directly to negotiate, ask questions, and send barter offers in real-time.

- 🛒 **Cart & Checkout** — Full cart management with quantity controls, multiple shipping options (Regular, Express, Same Day), and multiple payment methods (BCA VA, BRI VA, GoPay, QRIS, Indomaret, COD).

- 🏪 **Seller Dashboard** — Sellers get a full dashboard to manage listings, upload products, track incoming orders, monitor expedition status, and view buyer chats.

- 📦 **Purchases & Order Tracking** — Buyers can track their orders through status stages: Pending → Diproses → Dikirim → Selesai, and confirm receipt when delivered.

- 👤 **Profile & Shop Setup** — Users can set up a personal profile and shop, including shop name and branding, required before accessing the seller dashboard.

- 🔐 **Authentication** — Secure email-based sign-up and login via Supabase Auth, with protected routes for authenticated users.

---

## 🎯 Problem Solved

The fast fashion industry generates millions of tons of waste annually as clothing is discarded after only a few uses. ThriftVerse tackles this by:

- **Reducing waste** — Making it easy to resell and swap clothes that are still wearable.
- **Building community** — Turning transactional thrifting into a social experience through chats, ThriftVid, and barter.
- **Creating trust** — Offering a structured, verified platform compared to informal secondhand channels.
- **Promoting sustainability** — Encouraging reuse and reducing unnecessary new purchases through a barter-first mindset.

## ⚙️ Technology Stack

### 💡 Frontend

| Technology          | Purpose                                       |
|---------------------|-----------------------------------------------|
| _React 18_          | UI component library                          |
| _Vite 4_            | Build tool and development server             |
| _React Router DOM 6_ | Client-side routing and navigation           |
| _CSS Modules_       | Component-scoped styling                      |

### 🗄️ Backend & Database

| Technology          | Purpose                                       |
|---------------------|-----------------------------------------------|
| _Supabase_          | Backend-as-a-Service (Auth, Realtime, Storage)|
| _PostgreSQL_        | Relational database (via Supabase)            |
| _Supabase Realtime_ | Live chat and order status subscriptions      |

### 🏗️ Architecture

ThriftVerse follows a **client-server architecture**:
- **Frontend** handles all UI interactions and state management.
- **Supabase** acts as the backend, handling authentication, product data, order transactions, and real-time chat.
- **PostgreSQL** stores users, products, orders, barter offers, and chat messages.

## 🚀 How to Run Locally

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/thriftverse.git
cd thriftverse
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Configure Environment Variables

Create a `.env.local` file in the root directory and fill in your Supabase credentials:

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

> You can find these values in your Supabase project dashboard under **Settings → API**.

### 4. Set Up the Supabase Database

<!-- [Opsional: jika ada file SQL schema atau migration, sertakan langkah untuk menjalankannya. Contoh:] -->
<!-- Import the database schema from the `supabase/` folder into your Supabase project via the SQL Editor or Supabase CLI. -->

```bash
# If using Supabase CLI:
supabase db push
```

### 5. Start the Development Server

```bash
npm run dev
```

The app will be available at `http://localhost:5173`.

### 6. Build for Production

```bash
npm run build
```
